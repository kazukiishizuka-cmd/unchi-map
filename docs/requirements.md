# うんちマップ — 要件定義書

**プロジェクト名:** うんちマップ
**作成日:** 2026-04-17
**最終更新:** 2026-04-18
**ステータス:** v2.3 稼働中

---

## 1. プロジェクト概要

うんちをした場所を地図上に記録・共有できるスマホアプリ。
仲間内で使い、お互いの記録をマップ上で閲覧できる。

### 1.1 ゴール
- うんちの場所・日時・コメントを手軽に記録できる
- 連続記録（ストリーク）でモチベーション維持
- フレンド間で公開記録を共有して楽しむ

### 1.2 ユーザー
- 仲間内（〜数十人規模）
- App Store / Google Play にはリリースしない

### 1.3 配布方法
- Expo Go + EAS Update 経由
- URL: `exp://u.expo.dev/68cc9a89-62ab-413c-aae9-21a9829b3a5f?channel-name=default`

## 2. 技術スタック

| レイヤー | 技術 | 選定理由 |
|---------|------|---------|
| フロントエンド | React Native (Expo) + TypeScript | ストア不要配布（Expo Go）、TS学習、Web知識の応用 |
| バックエンド | Supabase (PostgreSQL + Auth + Storage) | 認証・DB・API・ファイルストレージが一体。無料枠で運用可能 |
| 地図 | Apple Maps (react-native-maps) | Expo Go互換。無料 |
| 配布 | EAS Update + Expo Go | URLを共有するだけで配布可能 |

## 3. 機能要件

### F1: うんち記録
- GPS で現在地を自動取得
- 日時は自動記録（記録ボタン押下時）
- 記録完了後、マップに💩ピンが追加される
- 記録画面は下からスライドインするモーダル

### F2: コメント
- 各記録にフリーテキストのコメントを付与可能
- 任意入力（空でもOK）

### F3: 公開 / 非公開切替
- **ユーザー単位**で公開・非公開を設定
- デフォルト: 公開
- 設定画面のトグルで切替
- 非公開ユーザーの記録はフレンドにも表示されない
- RLS で DB レベルで制御

### F4: マップ表示（メイン画面）
- Apple Maps 上に過去の記録を💩ピンで表示
- 自分の記録 + フレンドの公開記録を表示
- ピンタップで吹き出し表示（ニックネーム / 日時 / コメント）
- 吹き出しタップでプロフィールモーダル表示
- 現在地は極小の青いドットで表示
- 右上に🔄リフレッシュボタン

### F5: ストリーク（連続記録）
- 1日1回以上の記録で継続カウント
- 日付が変わる前に記録がなければリセット（0に戻る）
- 記録画面・サイドメニュー・プロフィールにストリーク日数を表示

### F6: ストリーク通知
- 毎日 18:00 に、その日まだ記録がない場合プッシュ通知（未実装）
- 通知の ON/OFF は設定画面から切替可能

### F7: ユーザー登録・認証
- **ID**（一意・検索用）+ **ニックネーム**（表示名）+ **パスワード** で登録
- ID はフレンド検索に使用
- ニックネームはマップ上の表示名・ピンの吹き出し等に使用
- ログイン状態はAsyncStorageで永続化

### F8: フレンド機能
- ID で他ユーザーを検索
- フレンド申請 → 相手が承認で相互フレンド成立
- フレンド一覧の表示
- フレンド一覧からプロフィールモーダル表示

### F9: フレンドの記録閲覧
- フレンドの**公開記録のみ**マップ上に表示

### F10: プロフィール
- アバター画像（任意の画像をアップロード。Supabase Storageに保存）
- ニックネーム・ユーザーID（設定画面から変更可能）
- 自己紹介（設定画面から編集可能）
- ピンの色（6色から選択）
- プロフィールモーダル: マップの吹き出しタップ or フレンド一覧タップで表示

### F11: マイ記録（履歴）
- 自分の記録を新しい順にリスト表示
- 住所表示（逆ジオコーディング）
- 日時表示（秒なし）
- 各記録に🗑削除ボタン

## 4. 画面構成

### 4.1 ナビゲーション構造

```
メイン画面（マップ常時表示）
├── 左上 ハンバーガー → カスタムドロワー（左からスライドイン）
│   ├── プロフィール（アバター・ニックネーム・ストリーク）
│   ├── 🗺 マップ
│   ├── 📋 マイ記録
│   ├── 👥 フレンド（未読申請バッジ付き）
│   ├── ⚙️ 設定
│   └── 🚪 ログアウト
│
├── 右上 🔄 → 記録リフレッシュ
│
└── 右下 FAB 💩 → 記録モーダル（下からスライドイン）
    ├── 現在地表示（住所）
    ├── コメント入力
    ├── ストリーク表示
    └── 記録ボタン
```

### 4.2 画面一覧

| 画面 | 種別 | 概要 |
|------|------|------|
| マップ画面 | メイン | 常時表示。💩ピン一覧。ハンバーガー・リフレッシュ・FAB |
| 記録モーダル | モーダル | 下からスライドイン。記録入力フォーム |
| メニュードロワー | カスタムドロワー | 左からスライドイン（自前Animated実装） |
| マイ記録 | 画面遷移 | 自分の記録一覧（住所・日時・コメント・削除） |
| フレンド管理 | 画面遷移 | フレンド一覧（タップでプロフィール）・検索・申請・承認 |
| プロフィールモーダル | モーダル | アバター・ニックネーム・自己紹介・ストリーク |
| 設定 | 画面遷移 | アイコン変更・アカウント編集・自己紹介・ピン色・公開設定・通知・ログアウト |
| ログイン | 画面 | ID + PW 入力 |
| ユーザー登録 | 画面 | ID + ニックネーム + PW 入力 |

## 5. 非機能要件

| # | 項目 | 仕様 |
|---|------|------|
| NF1 | プラットフォーム | iOS / Android（Expo Go + EAS Update 経由配布） |
| NF2 | ユーザー規模 | 〜数十人 |
| NF3 | 可用性 | 個人開発レベル。多少のダウンタイム許容 |
| NF4 | データ保全 | Supabase のバックアップに依存。記録データが消えないこと |
| NF5 | セキュリティ | 非公開ユーザーの記録は他ユーザーに一切表示しない。Supabase RLS で制御 |
| NF6 | コスト | 無料運用（Supabase無料枠） |
| NF7 | レスポンス | 記録操作は数秒以内に完了する体験 |

## 6. データモデル

### profiles
| カラム | 型 | 備考 |
|--------|-----|------|
| id | uuid | Supabase Auth 連携（PK） |
| user_id | text (unique) | 検索用ID |
| nickname | text | 表示名 |
| pin_color | text | マップ上のピン色（デフォルト #4A90D9） |
| streak_count | int | 現在のストリーク日数 |
| last_record_date | date | 最終記録日 |
| is_public | boolean | 公開設定（デフォルト true） |
| avatar_url | text | アバター画像URL |
| bio | text | 自己紹介 |
| created_at | timestamp | |

### records
| カラム | 型 | 備考 |
|--------|-----|------|
| id | uuid | PK |
| user_id | uuid | FK → profiles.id |
| latitude | float | 緯度 |
| longitude | float | 経度 |
| comment | text | 任意 |
| recorded_at | timestamp | 記録日時 |
| created_at | timestamp | |

### friendships
| カラム | 型 | 備考 |
|--------|-----|------|
| id | uuid | PK |
| requester_id | uuid | 申請者 |
| addressee_id | uuid | 被申請者 |
| status | text | pending / accepted / rejected |
| created_at | timestamp | |
