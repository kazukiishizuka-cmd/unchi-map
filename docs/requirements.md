# うんちマップ — 要件定義書

**プロジェクト名:** うんちマップ
**作成日:** 2026-04-17
**ステータス:** 要件確定

---

## 1. プロジェクト概要

うんちをした場所を地図上に記録・共有できるスマホアプリ。
仲間内で使い、お互いの記録をマップ上で閲覧できる。

### 1.1 ゴール
- うんちの場所・日時・コメントを手軽に記録できる
- 連続記録（ストリーク）でモチベーション維持
- フレンド間で公開記録を共有して楽しむ

### 1.2 ユーザー
- 仲間内（〜数十人規模）
- App Store / Google Play にはリリースしない

---

## 2. 技術スタック

| レイヤー | 技術 | 選定理由 |
|---------|------|---------|
| フロントエンド | React Native (Expo) + TypeScript | ストア不要配布（Expo Go）、TS学習、Web知識の応用 |
| バックエンド | Supabase (PostgreSQL + Auth) | 認証・DB・APIが一体。無料枠で運用可能 |
| 地図 | Google Maps API (react-native-maps) | 無料枠（月1万リクエスト）で十分 |
| 通知 | Expo Notifications | ストリーク通知用 |
| 配布 | Expo Go | アプリDLのみで利用可能。ビルド・審査不要 |

## 3. 機能要件

### F1: うんち記録
- GPS で現在地を自動取得
- 地図上でピン位置を手動修正可能
- 日時は自動記録（記録ボタン押下時）
- 記録完了後、マップにピンが追加される

### F2: コメント
- 各記録にフリーテキストのコメントを付与可能
- 任意入力（空でもOK）

### F3: 公開 / 非公開切替
- 各記録に公開・非公開フラグを持つ
- **デフォルト: 公開**
- 記録画面の右上にトグルボタンを配置し、即時切替可能
- 非公開記録は本人のみ閲覧可

### F4: マップ表示（メイン画面）
- Google Maps 上に過去の記録をピン表示
- 自分の記録 + フレンドの公開記録を表示
- ピンはユーザーごとに色・アイコンを分ける
- ピンタップで詳細表示（ニックネーム / 日時 / コメント）

### F5: ストリーク（連続記録）
- 1日1回以上の記録で継続カウント
- 日付が変わる前に記録がなければリセット（0に戻る）
- 記録画面にストリーク日数を表示

### F6: ストリーク通知
- 毎日 18:00 に、その日まだ記録がない場合プッシュ通知
- 「今日まだ記録してないよ！」的なリマインド
- 通知の ON/OFF は設定画面から切替可能

### F7: ユーザー登録・認証
- **ID**（一意・検索用）+ **ニックネーム**（表示名）+ **パスワード** で登録
- ID はフレンド検索に使用
- ニックネームはマップ上の表示名・ピンの吹き出し等に使用

### F8: フレンド機能
- ID で他ユーザーを検索
- フレンド申請 → 相手が承認で相互フレンド成立
- フレンド一覧の表示・削除

### F9: フレンドの記録閲覧
- フレンドの**公開記録のみ**マップ上に表示
- ピンの色・アイコンでユーザーを識別可能

### F10: 誰のうんちかわかる UI
- ユーザーごとにピンの色を自動割当
- ピンタップ時の吹き出しに**ニックネーム**を表示
- マップ上にフィルター機能（特定ユーザーの記録だけ表示）

## 4. 画面構成

### 4.1 ナビゲーション構造

```
メイン画面（マップ常時表示）
├── 左ドロワー（ハンバーガーメニュー）
│   ├── 📊 マイ記録（履歴一覧）
│   ├── 👥 フレンド管理
│   ├── 🔔 通知設定
│   └── ⚙️ 設定（アカウント情報等）
│
└── FAB（右下 💩+ ボタン）
    └── 記録ドロワー（左からスライドイン）
        ├── 現在地表示 + 位置修正
        ├── コメント入力
        ├── 公開/非公開トグル（右上）
        ├── ストリーク表示
        └── 記録ボタン
```

### 4.2 画面一覧

| 画面 | 種別 | 概要 |
|------|------|------|
| マップ画面 | メイン | 常時表示。ピン一覧。左上ハンバーガー、右下FAB |
| 記録ドロワー | ドロワー | 左からスライドイン。記録入力フォーム |
| メニュードロワー | ドロワー | 左からスライドイン。各種画面への遷移 |
| マイ記録 | 画面遷移 | 自分の記録一覧（リスト形式。日時・場所・コメント） |
| フレンド管理 | 画面遷移 | フレンド一覧・検索・申請・承認 |
| 設定 | 画面遷移 | 通知ON/OFF、アカウント情報、ログアウト |
| ログイン | 画面 | ID + PW 入力 |
| ユーザー登録 | 画面 | ID + ニックネーム + PW 入力 |

## 5. 非機能要件

| # | 項目 | 仕様 |
|---|------|------|
| NF1 | プラットフォーム | iOS / Android（Expo Go 経由配布） |
| NF2 | ユーザー規模 | 〜数十人 |
| NF3 | 可用性 | 個人開発レベル。多少のダウンタイム許容 |
| NF4 | データ保全 | Supabase のバックアップに依存。記録データが消えないこと |
| NF5 | セキュリティ | 非公開記録は本人のみ閲覧可。Supabase RLS で制御 |
| NF6 | コスト | 無料〜低コスト運用（Supabase無料枠 + Google Maps無料枠） |
| NF7 | レスポンス | 記録操作は数秒以内に完了する体験 |
| NF8 | 位置情報プライバシー | 非公開記録の位置情報は他ユーザーに一切公開しない |

## 6. データモデル（概要）

### users
| カラム | 型 | 備考 |
|--------|-----|------|
| id | uuid | Supabase Auth 連携 |
| user_id | text (unique) | 検索用ID |
| nickname | text | 表示名 |
| pin_color | text | マップ上のピン色 |
| streak_count | int | 現在のストリーク日数 |
| last_record_date | date | 最終記録日 |
| created_at | timestamp | |

### records
| カラム | 型 | 備考 |
|--------|-----|------|
| id | uuid | PK |
| user_id | uuid | FK → users.id |
| latitude | float | 緯度 |
| longitude | float | 経度 |
| comment | text | 任意 |
| is_public | boolean | デフォルト true |
| recorded_at | timestamp | 記録日時 |
| created_at | timestamp | |

### friendships
| カラム | 型 | 備考 |
|--------|-----|------|
| id | uuid | PK |
| requester_id | uuid | 申請者 |
| addressee_id | uuid | 被申請者 |
| status | text | pending / accepted / rejected |
| created_at | timestamp | |
